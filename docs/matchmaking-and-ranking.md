# マッチング＆ランキング設計

作成日: 2026-06-14 / 関連: [ADR-0001](adr/0001-rule-optimization-for-ranked.md)（ヘッズアップ＋捨て）

## マッチング

- **全モード ヘッズアップ（1対1）固定**。捨てありのデッキ供給上限（1人16枚×2=32 ≤ 52）による制約でもある
- **ランダムマッチは人間同士のみ**。CPU（ボット）とはマッチさせない（CPU相手はレートが動かず無意味なため）
  - カジュアル / レートで**別キュー**。同じ種別の人間が2人揃った瞬間にマッチ
  - 相手が見つからなければキューで**待機継続**（タイムアウトでのCPU補充はしない）。ユーザーはいつでもキャンセル可
  - CPUと遊びたい人は「CPU対戦」モード（完全オフライン）へ誘導
- **フレンド対戦**: 4桁の合言葉で1対1。招待リンクでも参加可
- **試合中の切断・放置（AFK）**: マッチングのCPU補充とは別物。試合が壊れないよう、配置制限時間（30秒）超過 or 切断猶予（45秒）切れで **CPUが代行配置**して試合を完走させる。切断した本人は負け扱いでレートが下がる（自己責任）

## レート

- 初期1000、Eloベース（[rating.ts](../packages/shared/src/rating.ts)）。**rated 対戦のみ変動**
- ボット（uid空）相手・カジュアル・CPU対戦は**レート不変**（水増し防止）
- ランク: ブロンズ / シルバー / ゴールド / プラチナ / ダイヤ / マスター

## ランキング更新タイミング

- **リアルタイム反映**: 試合が `game_over` になった時点で `ratingStore.recordResults` がレートと戦績を即更新し、ファイルへ永続化する
- **取得はプル型**: 戦績・ランキング画面を**開くたびに**サーバーへ `stats` / `ranking` を問い合わせて最新を表示（クライアントキャッシュなし）。常時ポーリングはしない
- **集計範囲**: 現状は**全期間・単一ラダー**（シーズンなし）。ランキングは上位20件＋自分の順位を返す
- **対戦履歴**: UIDごとに直近20試合を保持（勝敗・相手・レート変動・日時・モード）。戦績画面に表示

## 永続化（現状と課題）

- MVPは **JSONファイル**（`packages/server/data/ratings.json`、`OFC_RATING_FILE` で変更可）
- ⚠️ Render無料枠などはディスクが揮発し**再デプロイで消える** → 本番は永続ボリュームか外部DBへ。`ratingStore` のIF（getStats/getRating/recordResults/getTopPlayers/getRank）に閉じてあり差し替え容易

## 将来検討（未実装）

- シーズン制（期間リセット＋過去シーズンの殿堂）
- 長時間マッチしない場合の「CPU練習に誘導」または「待機中に通知でマッチ成立を知らせる」
- レート帯マッチング（近いレート同士を優先）。現状は到着順
- 不正・放置の常習者対策（連続切断のペナルティ強化）
