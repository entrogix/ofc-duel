# ADR-0001: レート戦に向けたルール最適化（ターボに「捨て」を導入し、ランクはヘッズアップ）

- **Status**: Accepted
- **Date**: 2026-06-14
- **Deciders**: ユーザー（プロダクトオーナー） / Claude
- **Related commit**: （未）

## Context

本アプリは参考ブログ（boardgameblog.net「オープンフェイス・チャイニーズポーカー・ターボ」）の
ルールをほぼ忠実に実装している（差異は1点のみ：バースト時にアプリは「6点＋相手の役ボーナス」を
支払う＝標準OFC寄り、ブログは「6点のみ」）。

レート/ランク戦を本実装したことで、2つの懸念が顕在化した。

1. **丸パクリ回避**：ルール自体は著作権保護の対象外（保護されるのは具体的な文章表現・図・名称）。
   よって法的に必要なのは「名称をブログ語から変更」「ブログの文章/表を転記しない」「著者へクレジット一報」。
   一方で“ブログの完全コピー”でないプロダクト独自性は、レート戦の差別化要素としても欲しい。
2. **レート戦の品質**：今のターボ（5-4-4・捨てなし・相手盤面を読めない同時公開）は
   **意思決定が極端に少なく運の比率が高い**。これだとレートがノイジーで、
   「簡単だけど奥深い」（ベンチマーク=動物タワーバトル）の“奥深い”が痩せる。

### 一般的OFC vs ブログ（＝現状）の差分

| 項目 | 一般的なOFC | ブログ＝現状 |
|---|---|---|
| 配り方 | 5枚→1枚ずつ×8（クラシック）／5→3配り2置き1捨て×4（パイナップル） | 5-4-4・全部置く・捨てなし |
| 公開 | オープンフェイス＝置くたび即公開（相手を読む） | 裏向き→ストリート毎に同時公開 |
| 手番 | 順番に1人ずつ | 全員同時 |
| 人数 | 2〜3人が競技標準 | 4人（3人版を別途） |
| ミドル最高ボーナス | ロイヤル50（ミドルはバックの2倍） | SF止まり30 |
| 試合終了 | 区切りなし | 持ち点50＋2巡 or 破産 |

ブログの独自性は **①ターボ配り（捨てなし）②裏向き→同時公開 ③4人＋持ち点制**。
このうち **②同時公開はオンラインで最良**（手番待ちゼロ・全員同時・ラグ耐性）であり、変えない。

## Decision（確定）

レート戦の深さと差別化を同時に満たすため、**ターボ配りに「毎ストリート1枚捨て」を導入し、全対戦をヘッズアップ(2人)固定**とする。

- **全対戦＝ヘッズアップ(2人)固定**（カジュアル・レート問わず）。4人/3人卓は廃止。実力が綺麗に出てマッチングも単純。
- 配り（確定）: `6枚配り→5枚置き1枚捨て` → `5枚配り→4枚置き1枚捨て` → `5枚配り→4枚置き1枚捨て`（計13枚配置・**毎ストリート1枚捨て＝3枚捨て**・16枚ドロー）
- **裏向き→同時公開は維持**（online最適）。
- **FL = 一律14枚配り→13枚置き1枚捨て**（突入も継続も14枚で固定）。突入条件=フロントQQ+、継続条件=現行通り（フロント3カード/ミドルFH+/バック4カード+）。継続率15%で報酬十分かつ非runaway（FL節の根拠）。
- 名称をブログ語（「ターボ」）依存から変更し、著者クレジットを明記。

### 配り枚数の決定根拠（シミュレーション 2026-06-14）

ヘッズアップ6,000ハンドのモンテカルロ（[completion_sim.ts](../../packages/shared/sim/completion_sim.ts)）で、捨て枚数と完成率の関係を比較：

| 指標 | 捨てなし | **1枚捨て** | 2枚捨て |
|---|---|---|---|
| バースト率 | 16.3% | **12.0%** | 11.6% |
| FL突入率 | 1.4% | **6.2%** | 16.6% |
| 平均ロイヤリティ | 0.99 | **2.59** | 5.43 |
| バック フラッシュ以上 | 7.1% | **15.8%** | 31.3% |

→ 捨てなしは「agency薄・FLほぼ無・地味」、2枚捨ては「役インフレ・FL頻発で逆に運寄り＆単調」。
**1枚捨てが運と実力のバランス最良**（FLが現実的目標6%、バーストは管理可能リスク12%）。

### 決定的な制約：52枚デッキの供給上限

捨てありは1人16枚必要。ヘッズアップ(2人)なら32枚で52枚以内に収まる。
3人=48、4人=64(破綻)。**ヘッズアップ固定なら供給に余裕があり、FLの追加配り分も吸収できる**（下記FL節）。

### FL（ファンタジーランド）の配り：継続率シミュレーション（2026-06-14）

[fl_continuation_sim.ts](../../packages/shared/sim/fl_continuation_sim.ts) で、FL中の配り枚数別に
継続率（フロント3カード/ミドルFH+/バック4カード+を継続最優先で組んだとき）を測定。

| 指標 | A: 13枚(捨てなし=現状) | B/C: 14枚(1枚捨て) |
|---|---|---|
| 継続率 p | 6.9% | 15.0% |
| FL平均ロイヤリティ | 8.9点 | 12.3点（通常≈2.6の約5倍） |
| 平均連続FL数 1/(1-p) | 1.07 | 1.18 |
| FL滞在割合(突入6.2%前提) | 6.2% | 6.8% |

**所見**: 一方的化リスクは両案ともほぼ皆無（継続15%でも平均1.18ハンドで止まる）。
一方A案(13枚)はFLの報酬が痩せすぎ。**継続率の観点からもFLは14枚（B/C）が妥当**。
プログレッシブ(C)は継続が常に14枚基準＝定常継続率15%で一定で、強い突入(KK/AA/トリプル=15〜17枚)が
**最初のFLハンドだけ**粘りと報酬を増す自己抑制的設計。runawayにはならない。
→ FL案の最終確定（B or C）は保留中だが、データはA案を否定し14枚ベースを支持。

## Consequences

### Positive
- 毎ストリートに「どれを捨てるか」の本物の判断が増え、**スキル天井が上がる＝レートが意味を持つ**
- ブログには無い要素＝**差別化＆丸パクリ回避**を一手で達成
- 操作は「置く＋1枚外す」だけ＝**手軽さは維持**
- 同時公開を保つので**オンラインの手番ゼロ**は不変
- ヘッズアップ・レートは分散が小さく、ラダーの信頼性が高い

### Negative / Trade-off
- ヘッズアップ固定で**3〜4人のグループ対戦を廃止**（カジュアルの大人数遊びは失う）
- **CPU AIに「捨て選択」ロジック**が必要（実装済み: `chooseCpuPlacementWithDiscard`）
- **FLの配り方**（13→14枚化）と捨てルールの整合（B/C確定が必要）
- エンジン/ビュー/テストの改修（`dealt`枚数 ≠ 配置枚数 を許す変更）
- 既存プレイヤーには配り方が変わる（学習コスト）

## Alternatives Considered

- **最小案：ルール不変・レートのみヘッズアップ化**
  低コスト低リスクで分散は下げられるが、差別化もスキル天井も上がらない。
  → レート品質の根本（運寄り）が残るため不採用。ただし「捨て」実装前の暫定段としては有効。
- **全面パイナップル化（5→3配り2置き1捨て×4）**
  標準に寄りすぎて差別化が弱く、4人不可、かつターボの“速さ”が薄れる。 → 不採用。
- **オープンフェイス即時公開で深さを足す**
  読みの層は増えるが、**オンラインの同時性を壊す**（手番待ち発生）。 → 不採用。

## 確定済み（旧Open Questions）

1. ✅ 捨ての回数 → **毎ストリート1枚（計3枚捨て・6/5/5配り）**（シミュレーションで決定）
2. ✅ 人数 → **全モード ヘッズアップ(2人)固定**。3〜4人卓は廃止
3. ✅ FLの扱い → **一律14枚配り・1枚捨て**（継続率シミュレーションで決定）
6. ✅ レート人数 → ヘッズアップ固定（上記2に統合）

## 残Open Questions（別途・実装と並行で可）

4. ロイヤリティ改定（ミドルRF=50へ戻す等）— 別ADR候補
5. 名称候補（「ターボ」を外した独自名）とブログ著者へのクレジット文面
7. プログレッシブFL（強い突入で最初のFLを15〜17枚に）— 将来の差別化/シーズン機能候補。今回は一律14枚で先行
