# OFCターボ スマホアプリ — 要件定義

作成日: 2026-06-11
元ネタ: http://boardgameblog.net/2022/06/02/ocpt/ （オープンフェイス・チャイニーズポーカー・ターボ）

## ユーザー確認済み要件

| 項目 | 決定内容 |
|---|---|
| スコープ | フルゲーム（配布・配置・役判定・採点まで全てアプリ内完結） |
| 対戦形式 | CPU対戦 ＋ オンライン対戦の両方 |
| ルール | 通常OFCターボ（5-4-4配布、2〜4人） |
| 技術 | React Native / Expo |

## 実装側の設計判断（デフォルト採用）

- **オンライン対戦はルームコード方式**: 4文字の合言葉でルーム作成/参加。マッチングプール・アカウント機能は持たない（v1）
- **サーバー**: Node.js + ws（WebSocket）。ゲームロジックはサーバー権威（authoritative）
- **モノレポ**: `packages/shared`（純粋TSのゲームロジック、依存ゼロ）を app / server 双方から参照
- **ナビゲーションライブラリ不使用**: 画面数が少ないため state ベースの画面切替で依存を最小化

## ゲームルール仕様（ブログ記事準拠）

### 基本
- ジョーカー抜き52枚、2〜4人
- 13枚を フロント(3) / ミドル(5) / バック(5) に配置
- 役の強さが フロント ≦ ミドル ≦ バック を満たさないと**バースト**（同等は許容）
- フロントの役はハイカード/ワンペア/スリーカードのみ（ストレート・フラッシュなし）

### 配布（ターボ）
1. 5枚配布 → 全員同時に裏向き配置 → 全員完了で同時公開
2. 4枚配布 → 同様
3. 4枚配布 → 同様（計13枚）
- 一度置いたカードは移動不可

### 採点（ハンド終了時、全ペア間で精算）
- 行ごとに比較し、負けた行1つにつき相手へ1点支払い
- 同一相手に3行全敗 = 6点（スクープ、倍額）
- 同じ役で数字も同じなら引き分け（点数移動なし）
- バーストした人は、バーストしていない各相手に6点支払い ＋ 相手のロイヤリティも支払う（自分のロイヤリティは無効）
- 双方バーストなら点数移動なし
- ロイヤリティはペア間で差額精算

### ロイヤリティ
| 役 | フロント | ミドル | バック |
|---|---|---|---|
| ペア66〜AA | 1〜9点 | — | — |
| スリーカード222〜AAA | 10〜22点 | 2点 | — |
| ストレート | — | 4点 | 2点 |
| フラッシュ | — | 8点 | 4点 |
| フルハウス | — | 12点 | 6点 |
| フォーカード | — | 20点 | 10点 |
| ストレートフラッシュ | — | 30点 | 15点 |
| ロイヤルフラッシュ | — | 30点 | 25点 |

### ファンタジーランド（FL）
- **突入**: バーストせず、フロントがQQペア以上（スリーカード含む）
- **進行**: 次ハンドで13枚を一括配布。他プレイヤーが5-4-4で進める間に自由なタイミングで配置。FL中の盤面は精算まで非公開
- **継続**: フロントがスリーカード / ミドルがフルハウス以上 / バックがフォーカード以上 のいずれか
- **FL対象者がいる間はディーラー移動なし**

### ゲーム終了
- 各プレイヤー初期チップ50点（ゼロサム精算）
- 誰かが破産（チップ0以下）するか、各プレイヤーが2回ずつディーラーを務めたら終了
- 最終チップ最多が勝者

## 画面構成（v1）

1. **ホーム**: CPU対戦 / オンライン対戦 / ルール
2. **ゲーム画面**（CPU・オンライン共通）: 自分の3行ボード＋手札トレイ（タップで選択→行タップで配置）、相手のミニボード、確定ボタン
3. **精算モーダル**: ペアごとの行勝敗・ロイヤリティ・FL突入/継続の内訳
4. **オンラインロビー**: ルーム作成（コード表示）/ コード入力で参加 / 参加者一覧 / 開始

## v1の既知の制限

- オンライン対戦の再接続は未対応（切断するとそのハンドは続行不可）
- CPU AIはヒューリスティック（全数列挙＋盤面評価関数）。バースト回避優先の堅実型
- Pineapple（3人版）は未実装（設定切替の余地は engine に残す）
