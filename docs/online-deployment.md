# オンライン対戦の実現方法

作成日: 2026-06-13

## 現在の仕組み

```
スマホA ──┐
スマホB ──┼─ WebSocket ──> 対戦サーバー (packages/server, Node.js + ws)
スマホC ──┘                 ・ルームコード（4文字）でルームを管理
                            ・ゲームロジックはサーバー側で実行（サーバー権威 = チート不可）
                            ・クライアントには各自に見える情報だけ送信
```

- アカウント不要。「ルーム作成」→ 4文字コードを友達に伝える → 「参加」だけ
- サーバーは `npm start` で起動（ポート8787、`PORT`環境変数で変更可）

## 実現方法は3段階ある

### レベル1: 同じWi-Fi内（今すぐ・無料）

家族や同じ部屋の友達と遊ぶならこれで十分。

1. PCで `cd packages/server && npm start`
2. PCのIPアドレスを調べる（`ipconfig` → 例 192.168.1.10）
3. スマホアプリの「サーバーURL」に `ws://192.168.1.10:8787` を入力

制限: 同じネットワークにいる人としか遊べない。

### レベル2: クラウドにデプロイ（推奨・遠くの友達と遊べる）

**Render（無料枠あり）が一番簡単。** サーバーは素のNode.jsなので特別な対応は不要。

1. https://render.com にGitHubアカウントで登録
2. New → Web Service → このリポジトリを接続
3. 設定:
   - Root Directory: `products/ofc-turbo/packages/server`
   - Build Command: `npm install`
   - Start Command: `npm start`
   - Instance Type: Free
4. デプロイ完了で `https://ofc-turbo-server.onrender.com` のようなURLが発行される
5. アプリのサーバーURL欄に **`wss://`**（sを付ける）で入力:
   `wss://ofc-turbo-server.onrender.com`
   （TLSはRenderが終端するのでコード変更不要。RenderはPORT環境変数を渡してくる → 対応済み）

**無料枠の注意**: 15分アクセスがないとスリープし、次の接続に30秒〜1分かかる。
遊ぶ前に誰かが一度つなげば起きる。常時起動したければ $7/月。

代替: Fly.io（無料枠+WebSocket対応）、Railway（$5クレジット/月）。

### レベル3: ストア公開時の本格運用

- サーバーURLをアプリに埋め込み（入力欄は上級者向けに残す）
- 再接続対応（現状は切断＝ゲーム中断。トークンで席を保持する改修が必要）
- 不特定多数とのマッチングをするなら、アカウント/レート/マッチングプールの設計が必要
  → v1は「友達とコード共有」のままで十分。需要が出てから検討

## まとめ

| 方法 | 費用 | 手間 | 遊べる範囲 |
|---|---|---|---|
| 同じWi-Fi | 0円 | 5分 | 同じネットワーク内 |
| Render無料枠 | 0円 | 30分 | 世界中（スリープあり） |
| Render有料 | $7/月 | 同上 | 世界中（常時起動） |
